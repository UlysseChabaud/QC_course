Notions of fault tolerant quantum computation (04/12/23)

In this lecture we introduce the notion of fault tolerance for quantum hardware. So far we have studied error correction assuming that all components except possibly for the qubits hosting the logical information function perfectly. Fault tolerance refers to the techniques that allow to successfully carry out a computation even in the presence of other faulty components. These can include measurements, the auxiliary qubits used for syndrome extraction and logic gates.

After discussing some of the ingredients necessary to fault tolerance we will prove a special case of the celebrated threshold theorem, which shows that properly designed schemes allow to perform arbitrarily long computations with arbitrarily low logical error rates, provided the physical failure rate is below a certain finite value. The price to pay is that additional resources need to be used in order to keep logical information protected.

In the special case we consider, we make the simplification of considering stochastic noise models (see below) and a specific design based on concatenated codes. Although this is not necessarily the most realistic model or the most viable approach from a technological standpoint, it is very useful to gain some insight on the main mechanisms behind fault tolerance.

The need for fault tolerance

In our discussion so far, we have implicitly assumed that

  1. Errors only occur during “memory” regions, i.e. when quantum operations or error correction are not being performed and ancilla qubits are error free, and
  1. The quantum gates themselves do not induce any systematic errors within the logical data block.

This is quite unrealistic. When these hypotheses are relaxed, error correction needs to be performed in a specific way to still be effective. The additional techniques required are known under the name of fault tolerance.

In a nutshell, fault tolerance is possible if the physical error rate is low if we can do two things:

  1. construct families of codes that can correct errors of increasing weight and
  1. perform all required operations on encoded data, including error recovery routines, in such a way that the weight of errors does not grow too much.

We start by detailing a possible way to achieve the first part.

The ingredients for fault tolerance
Concatenation

A particular way to construct families of increasingly resilient codes is concatenation.

Concatenation is where encoded qubits are further encoded. In other words, logical qubits are re-interpreted as physical qubits in the construction of another error correcting codes. This forms a second level encoded qubit. We have already seen this procedure as a way of constructing Shor’s 999﻿ qubits code as a concatenation of the bit-flip and phase-flip codes.

In theory, this procedure can be repeated arbitrarily many times, even with different codes at each iteration. Not all codes, however, will be equally suited for fault tolerance, as we will see shortly.

Here, we focus on the [[7,1,3]][[7, 1, 3]][[7,1,3]]﻿ Steane code, the CSS code constructed from the [7,4,3][7,4,3][7,4,3]﻿ classical Hamming code. Starting with a group of un-encoded qubits (aka level-000﻿ logical qubits), we first encode into level-111﻿ logical qubits, each containing a block of seven. Each of these logical qubits are now protected with a distance-333﻿ quantum code. If we now take seven of these level-111﻿ logical qubits are perform the same encoding operation (using valid logical operations for the level-111﻿ qubits) we form a single level-222﻿ logical qubit. This is now a block containing 494949﻿ physical qubits and has a code distance of 32=93^2 = 932=9﻿.

A level-222﻿ logical qubit can therefore be faithfully recovered if four or less errors occur on any of the 494949﻿ physical qubits. For ggg﻿ levels of concatenation, the encoded state will consist of 7g7^g7g﻿ physical qubits and have a code distance of 3g3^g3g﻿ , allowing for the correction of 3g−12\frac{3^g−1}{2}23g−1​﻿ individual errors. Hence for a concatenation scheme, both the number of physical qubits required and the number of errors it can correct scale exponentially (with the number of qubits growing faster than the number of correctable errors).

Fault-tolerant circuit elements

Error propagation is the main reason why concatenation alone is not enough to perform arbitrarily accurate computations, and we also need to design our procedures in a fault tolerant fashion. Without error propagation, the errors would be confined to their original code block and reducing the time between correction rounds would suffice to ensure that the probability of uncorrectable errors decreases.

Threshold theorem

The threshold theorem is a consequence of fault tolerant circuit design and the ability to perform dynamical error correction (i.e. syndrome extraction and recovery operations). In principle, different threshold theorems exist depending on the noise model, error correcting code, architectural designs etc.

Rather than present a detailed derivation of the theorem for a variety of noise models, we will instead take a very simple case where we utilize a quantum code that can only correct for a single error, using a model that assumes uncorrelated errors on individual qubits.

Overhead

There is one point we need to be careful about: we have successfully reduced the error rate, but at the cost of compiling an initially simple circuit into many more gates. If the scaling of added gates vs reduced noise is too unfavourable we risk losing all of the advantage we expected in using a quantum computer in the first place.
