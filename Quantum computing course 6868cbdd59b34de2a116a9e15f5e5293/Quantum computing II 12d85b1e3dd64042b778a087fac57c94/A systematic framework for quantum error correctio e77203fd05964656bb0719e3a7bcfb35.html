A systematic framework for quantum error correction (21/11/23)

In the previous lecture we discussed classical linear codes and saw how the ideas of parity checks and syndrome decoding can be used to construct a first example of quantum error correcting code: Shor’s code. This code uses 9 physical qubits to protect a single logical qubit from arbitrary bit-flip or phase-flip errors. Shor’s construction contains many of the fundamental ideas underlying quantum error correcting codes (QECC), but it is not obvious how to generalize it.

We would like to go beyond the level of intuition and start organizing the ideas seen last time in a theoretical framework. This is also necessary to devise more systematic ways of constructing codes and benchmarking their efficacy.

As usual we can imagine that the two-step process of syndrome extraction and then correction is replaced by a coherent conditional operation. This is instructive because it clarifies the role of the auxiliary system. Imagine for simplicity that the code block starts in a pure state. Noise causes it to become entangled with the environment. The KL conditions prevent the entanglement from leaking information about the physical state to the environment but entropy in the data block increases nonetheless. When we apply the recovery map we are effectively transferring these correlations from the data block to the ancillary system.

The above results do not rely on the data carriers being composed of qubits and therefore hold in general.

Some general results on quantum codes
Distance of a qubit code

We will now focus on binary codes, namely those that can be formulated in terms of qubits. In this case, a sensible choice is to benchmark the effectiveness of a QECC by stating the maximum weight of Pauli strings that the code can correct against. Besides the parallelism with classical linear codes, this is motivated by two observations:

  1. from our previous discussion, we can assume that low-weight Pauli strings are leading-order errors in many cases of interest, even if they might not correspond exactly to the Kraus operators of the true error channel;
  1. since Pauli strings are a basis for the operators on n qubits, we can use them to expand the Kraus operators of any noise channel. In particular, we will be able to describe any channel whose action can at least approximately be described as independent action on small sets of qubits using low-weight Pauli strings.

Therefore we introduce the following definition of distance of a quantum code.

By our definition of distance, a code can correct ttt﻿ errors provided it has distance d≥2t+1d\geq2t+1d≥2t+1﻿. Note the resemblance with the correctable errors for classical linear codes of distance ddd﻿. The main difference in this case is that now we have different types of errors, besides bit-flip.

It is also possible to restrict the definition to only XXX﻿- or ZZZ﻿-type errors, namely Pauli strings whose non-trivial tensor components are either XXX﻿ or ZZZ﻿ operators. Sometimes distances for special kinds of errors are indicated as dXd_XdX​﻿ or dZd_ZdZ​﻿ for example, and it might happen that these two numbers do not coincide.

This was the case for example for the three-qubit bit-flip code, which has dX=3d_X=3dX​=3﻿ and dZ=1d_Z = 1dZ​=1﻿, so that it can correct any weight-111﻿ XXX﻿-type Pauli error but no ZZZ﻿-type error.

On the other hand, one can verify that Shor’s code satisfies the KL conditions for all weight-1 Pauli strings and therefore has distance 3 and can correct any error that can be written as linear combination of Pauli matrices acting on a single qubit, including e.g. X1+X2X_1+X_2X1​+X2​﻿ or X1Z1+Z3X_1Z_1+Z_3X1​Z1​+Z3​﻿, which themselves act non-trivially on two qubits. This is once again a consequence of error digitization: we can devise a measurement that collapses the corrupted state to one of the error subspaces and then apply a suitable unitary correction.

Code degeneracy and the quantum Hamming bound

In coding theory it is often desirable to introduce notions of optimality, so that we can evaluate how good a specific code is with respect to what is possible in principle. In order to do that, we might want to ask questions such as:

“what is the minimum number of physical qubits required to protect kkk﻿ logical qubits from arbitrary single-qubit errors?”

Since we have seen that we can construct quantum codes in a similar way as classical linear codes exploiting parity check equations (and we will see a systematic way to do so shortly), we might hope that some of the proof techniques used in the classical case still work for quantum codes.

Shor’s code however already shows that there are fundamental difference between quantum and classical codes: since it can correct up to 1 arbitrary single-qubit error, if it behaved like a classical code we would expect Shor’s code to assign a unique syndrome to all such errors. However this is not the case. In fact, applying any ZZZ﻿ Pauli on any qubit in the same block leads to the same state, and hence the same syndrome. This is an example of code degeneracy:

The upside is that degenerate codes can, in a way, be “more efficient” than classical linear codes, in the sense that they can correct more errors than they can uniquely identify. On the other hand, many classical results will need additional qualifiers in order to be exported to the quantum case.
