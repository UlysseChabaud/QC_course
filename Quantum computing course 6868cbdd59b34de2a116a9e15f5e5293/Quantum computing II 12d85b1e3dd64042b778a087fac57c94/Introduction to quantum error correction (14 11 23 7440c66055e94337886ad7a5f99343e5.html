Introduction to quantum error correction (14/11/23)

Error correction and fault tolerance

In order to process information, we need to encode it on some physical system. No system is ever completely controlled and isolated from its environment. Uncontrollable interactions with the environment and potentially defects of the system itself can cause information to become corrupted. This is especially true for quantum information processing (QIP), where the systems are inherently more fragile and harder to completely isolate from their environment.

Moreover, to benefit from the advantages that QIP theoretically offers over classical information processing, we need to protect the delicate properties arising from quantum superposition. Actual physical noise is mostly continuous, meaning that arbitrarily low perturbations can happen. Classical digital encodings are designed so as to enforce a discreteness (e.g. physical bits having either the value 0 or 1 at any given point) which makes classical information processing very robust to noise. Despite this fact, a number of applications even in the classical domain still require some form of error correction in order to be carried out without faults. Typical examples include transmissions over wireless networks or trivially using backups for data storage. The latter case can be seen as a form of repetition code, which we will see in more detail below. The underlying intuition, which can be extended to virtually any error correction scheme, is that one can contrast noise at the cost of adding some form of redundancy to the encoded information. Besides providing a backup in case something goes wrong, the additional resources can often be designed in such a way that we can, with high probability, figure out if something went wrong and correct potential errors.

The prominence of information transmission and processing in modern times has stimulated a huge amount of research in error correction for classical and, more recently, quantum applications. Remarkably, despite the great variety of information processing systems, a unified theory can be developed that allows to understand general properties of codes at an abstract, mathematical level. The aim of this course is to give an introduction to this high-level picture, generally known as coding theory, with emphasis on the quantum case, and to provide you with a vocabulary which should enable you to understand more specialized literature.

A related, but separate (and harder) problem is to keep information protected in the course of a computation when some other parts besides the data carriers can be faulty, such as logic gates and read-outs. This property is generally known as fault tolerance.

The relevance of fault tolerance in the quantum case is easy to understand if one thinks that provable computational advantage in the quantum case is mostly only obtained in the asymptotic regime of larger inputs and problem size, which entails applying more gates in the course of the algorithm. If we fix the highest error probability we can accept in the outcome of a computation, we intuitively understand that if components have a constant error rate we are quickly limited in the size of the problems we can work with or the length of the computation, if we naively run the algorithm without error correction. Indeed, the probability of no error occurring drops exponentially with the number of components. Therefore we have to conclude that technological progress lowering the component fault-rate, although necessary, cannot by itself be the long term solution, as it might change the overall error rate but not its exponential behaviour. Luckily, a remarkable and deep result states that using a appropriate error correction schemes it is theoretically possible to run arbitrarily long computations with arbitrarily low error rate, even if each component has a fixed probability of being faulty, provided this probability is below some threshold. This result that we will see at the end of our course is therefore known simply as the threshold theorem.

Decoding as inference problem

In practice, we typically do not have a guarantee that the errors have strictly bounded weight. Rather, we often have a statistical description of the noise process and in many cases low-weight errors are more likely, as in the case of the binary symmetric channel. Therefore error correction only succeeds with some finite probability, and the goal becomes, given a noise channel, to find codes that

  1. introduce low overhead (i.e. whose rate is as high as possible);
  1. admit a decoding strategy that succeeds with high probability and can be computed efficiently (i.e. with an algorithm whose runtime is at most polynomial in the size of the input).

Remarks

A few remarks are in order.

  1. First, the syndrome extraction scheme for Shor code entails measuring a total of 888﻿ operators in order to extract the full syndrome. Therefore 888﻿ bits of information are extracted from the system, which is consistent with our intuitive lower bound of 444﻿.
  1. Since the procedure for detection and correction of XXX﻿errors does not interfere with X-type checks, we can deal with each case separately: for example, we can perform syndrome extraction for bit-flips and the measurements will be independent of any ZZZ﻿ errors present in the input state, as all measured observables commute with any ZZZ﻿ operators. We can then correct the deduced XXX﻿operators without introducing further phase-flips and when this whole procedure is finished we can turn to syndrome extraction and correction for ZZZ﻿-type errors. This is a general property of so-called Calderbank-Shor-Steane (CSS) codes, of which Shor code is a representative, and that we will see in more detail in the next lecture. In particular, this means that we can also correct errors of the form ρ↦ZjXjρXjZj\rho\mapsto Z_jX_j\rho X_jZ_jρ↦Zj​Xj​ρXj​Zj​﻿, such as those appearing in EXZ\mathcal{E}_{XZ}EXZ​﻿.
  1. We have seen that different errors could lead to the same syndrome pattern, but it did not matter as they had exactly the same effect on the encoded state. This is also a property that we will see in more detail in future lectures and is known as code degeneracy.
Error digitization

So far we have assumed that errors happen either deterministically or, in the worst case, are sampled from a classical probability distribution. However, error themselves can occur in superposition. Let us go back to the bit-flip code for simplicity and consider the case of the error introduced by a unitary evolution

Remarkably, we do not need to do any extra work to correct for this with respect to what we have already seen. Indeed, if we measure Z1Z2Z_1Z_2Z1​Z2​﻿ we will either collapse the state to ∣ψ⟩|\psi\rangle∣ψ⟩﻿ if we get outcome zero, or to X1∣ψ⟩X_1|\psi\rangleX1​∣ψ⟩﻿ if we get outcome one. Either way, we will not know about the superposition that might have existed before the measurement but we will be able to bring the system back to the correct after measuring Z2Z3Z_2Z_3Z2​Z3​﻿ and applying the adequate correction as in the deterministic error case.
